**CHATHU MD** is a production-grade, multi-session WhatsApp automation platform built on
`@whiskeysockets/baileys`, shipping with a **Cyber-Glass Admin Dashboard**, a rich command
library, a full automation runtime (auto-reply rules, scheduler, broadcast, anti-link /
anti-spam / anti-bad-word protections), and a pro-level security + DevOps stack.

This `Pro Edition` release (v2.3.0) lands all security, runtime, UI and DevOps
hardening in one drop — it is the recommended line for self-hosted or VPS deployments.

## ✨ What's new in Pro Edition (v2.3.0)

### 🔐 Security
- **bcrypt-hashed `ADMIN_PASS`** (auto-detects `$2a$` / `$2b$` / `$2y$`; plaintext falls
  back to constant-time compare for local dev). Helper: `npm run hash-pass -- '<pwd>'`.
- **Timing-oracle-safe** password verification — both buffers are padded to the same
  length before `crypto.timingSafeEqual`, so response time no longer leaks password
  length.
- **Login rate-limit** (5 attempts / 5 min, 2 min cooldown) + constant-time username
  compare.
- **Audit log** persisted to `db.json` (250-entry ring). Every mutating `/bot-api/*`
  call (and every auth event) is captured with method, path, user, IP, status.
  New `GET /bot-api/audit?limit=N` endpoint.
- **Safer dashboard defaults** — `DEFAULT_ADMIN_PASS` / `DEFAULT_JWT_SECRET` fallbacks
  for first boot, with a loud warning in the logs when either is still at its default.
- **Safer uncaughtException handling** — known-noisy Signal / decryption errors are
  swallowed, but any other uncaught exception now logs and `exit(1)`s (Node state is
  undefined after an uncaught; a process manager should restart).

### 🛡️ Runtime hardening
- **Exponential reconnect backoff** for session-manager (5s → 10s → 20s → 40s → 80s →
  cap 120s; resets on successful relink).
- **Single-flight command dispatch** — the two-loop message pipeline shares a
  `skippedMessageIds` set so anti-link / anti-spam / mute / private-mode / status-
  broadcast decisions are never double-processed.
- **Chat-scoped deduplication** — message-id dedup key is `${remoteJid}:${msg.key.id}`
  with a single-interval sweep (instead of thousands of `setTimeout` callbacks), so
  WhatsApp's non-globally-unique message ids can't silently drop a legitimate message
  in chat B just because chat A saw the same id earlier.
- **Bounded spamMap** — the per-sender anti-spam table is swept periodically so it
  doesn't leak memory across a long-running bot.
- **Anti-bot-loop filter** is now prefix-aware — `.ai explain chathu md` and other
  legitimate commands whose arguments contain `chathu md`, `generated by`, etc. are
  no longer silently dropped.
- **Non-mutating message watermark** — `lib/message-manager.send()` now watermarks a
  copy of the payload, so reused template objects can't accumulate zero-width spaces.

### 🖥️ Admin UI polish
- **Light/dark theme toggle** persisted to `localStorage.chmd_theme`
  (keyboard shortcut: **Ctrl/Cmd + Shift + L**).
- **Command palette** — **Ctrl/Cmd + K** (or `/` when nothing is focused). Fuzzy
  jumps across Dashboard / Sessions / Users / Groups / Commands / Broadcast /
  Auto-Reply / Scheduler / Files / Settings / Logs, plus actions like Toggle Theme,
  Restart Bot, Sign Out, Send Test Message.
- **Live sysmetrics chart** on the dashboard (CPU% + memory% over the last 60
  samples, refreshing every 5 s, theme-aware colors).
- **Send Test Message** modal globally available (palette + Sessions page button).

### 🚢 DevOps
- **Multi-stage `Dockerfile`** (`node:20-bookworm-slim` deps stage → slim runtime
  stage). Drops dev tools + Python build deps from the final image; runs through
  `tini` for proper signal handling; `HEALTHCHECK` polls `/bot-api/health` every 30s.
- **`.dockerignore`** excludes `node_modules`, sessions, `db.json`, logs, IDE
  configs, attachments, etc. — meaningfully smaller build context.
- **`/bot-api/health`** public endpoint: `200` with `{ ok, status, number, uptime,
  memUsedMB, version, node }` when connected/connecting/idle/awaiting-link, `503`
  otherwise. Docker `HEALTHCHECK` / Render / Fly readiness probe ready.
- **`scripts/syntax-check.js`** — walks every `.js` in the repo and runs
  `node --check`. Currently **49 / 49 files** clean. Exposed as
  `npm run syntax-check`.
- **GitHub Actions** workflow (`.github/workflows/ci.yml`) gates every push to
  `master` and every PR on `npm ci` + `npm run syntax-check`.

## 💎 Feature summary

- **🌐 Cyber-Glass Admin Panel** — real-time dashboard with glassmorphism aesthetics
  and 12 pages: Dashboard, Sessions, Users, Groups, Commands, Broadcast, Auto-Reply,
  Scheduler, Files, Logs, Settings, plus the Command Palette overlay.

  Sri Lankan) or QR scan.
- **🧠 AI auto-reply** — Gemini / OpenRouter / Groq fan-out with persona + language +

- **🛡️ Global protections** — Anti-Link (with bot-admin awareness), Anti-Spam
  (bounded), Anti-Bad-Word (with bot-admin awareness), Mute, Work-Mode (public /
  private / self). All honoured by both the protection loop and the dispatch loop.
- **🌸 89+ commands** across Media (YouTube, Instagram, TikTok, Facebook, etc.),
  Search, Utility, Fun, Group admin, AI, Economy, Anime, NSFW (toggle-gated).
- **📡 Broadcast manager** — send announcements to all groups/users with history
  tracking and audit entries.
- **🧩 Automation runtime** — auto-reply rule engine + scheduler with persistent
  `db.json` storage.
- **🔄 Auto-self-healing** — built-in anti-crash loop that recovers transient
  Baileys/Signal noise in under 10 s, plus structured exit on anything else so a
  process manager can restart cleanly.

## 🚀 Quick Start

### 📋 Prerequisites

- **Node.js**: 20.x (18.x may work but is not tested in CI)
- **Git**

### ⚙️ Installation

```bash
git clone https://github.com/chathura4003-blip/CHATHU-V3.git
cd CHATHU-V3
npm ci
npm run syntax-check   # sanity check (expect 49/49 OK)
npm start
```

Open **`http://localhost:5000`** in your browser to reach the dashboard. First-boot
defaults live in `.env.example` — copy to `.env` and adjust.

### 🔑 Hash your admin password (recommended)

```bash
npm run hash-pass -- 'your-strong-password'
# copy the printed $2b$...  hash into ADMIN_PASS in .env
```

Login response reports `passwordHashed: true` when you're on bcrypt — use that to
verify your deployment.

### 🩺 Health check

```bash
curl -s http://localhost:5000/bot-api/health | jq
```

Returns `200` with status/uptime/version when the bot is connected/awaiting link,
`503` otherwise. Suitable for Docker `HEALTHCHECK`, Render / Fly readiness probes,
or any external uptime monitor.

### 📜 Audit log

```bash
curl -sH "Authorization: Bearer $TOKEN" \
     'http://localhost:5000/bot-api/audit?limit=25' | jq
```

## 🐳 Docker

```bash
docker build -t chathu-md:v2.3.0 .
docker run -d --name chathu-md \
  --restart unless-stopped \
  -p 5000:5000 \
  -e ADMIN_USER=admin \
  -e ADMIN_PASS='$2b$12$...hash...' \
  -e JWT_SECRET='<long-random>' \
  -v chathu-md-data:/app \
  chathu-md:v2.3.0
```

The image uses a multi-stage build with `tini` for PID 1, drops Python / build
dependencies from the final layer, and includes `HEALTHCHECK --interval=30s` that
polls `/bot-api/health`. A process manager (Docker `--restart`, Fly, Render, pm2 or
systemd) is expected — the bot will intentionally `exit(1)` on a non-noisy uncaught
exception rather than continue in an undefined state.
